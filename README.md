# אריאל: משלב הפוקימונים 🎮

משחק WEB אסטרטגי (Auto-Battler / Merge) מותאם למובייל, בהשראת Merge Tactics ו-TFT, עם
דמויות פוקימון. בונים צבא, ממזגים פוקימונים, מניחים אותם אסטרטגית על הלוח, ולוחצים "שחק"
כדי להילחם אוטומטית מול היריב. ניצחון → מטבעות + גלגל מזל + שלב הבא.

## מסמכי תכנון
- [docs/GDD.md](docs/GDD.md) — מסמך עיצוב המשחק המלא.
- [docs/BACKLOG.md](docs/BACKLOG.md) — רשימת משימות הפיתוח.

## סטאק טכנולוגי
- **React 18 + TypeScript + Vite** — ליבת האפליקציה.
- **Zustand** — ניהול מצב.
- רינדור לוח/קרב מבוסס DOM/CSS (mobile-first, RTL), ללא תלות בשרת.
- **PokeAPI sprites** — תמונות הפוקימונים (הורדה מקומית, ר' למטה).

## התקנה והרצה
```bash
npm install
npm run fetch-sprites   # מוריד את תמונות הפוקימונים ל-public/assets/pokemon
npm run dev             # שרת פיתוח (http://localhost:5173)
```

### סקריפטים נוספים
```bash
npm run build      # בנייה לפרודקשן + בדיקת טיפוסים
npm run preview    # תצוגה מקדימה של הבנייה
npm run balance    # בדיקת איזון: מדידת אחוזי ניצחון לפי שלב מול היעדים ב-GDD
```

## מבנה הפרויקט
```
src/
  data/         # מאגר הפוקימונים (pokedex) + טבלת יתרונות טיפוסים
  game/         # לוגיקת ליבה: סטטיסטיקות, כלכלה, שלבים, AI/קושי, מנוע קרב, גלגל מזל
  store/        # Zustand store (מצב המשחק + פעולות + שמירה ל-LocalStorage)
  screens/      # מסכים: פתיחה, הכנה, קרב, תוצאה, גלגל מזל
  components/   # קומפוננטות משותפות (ספרייט פוקימון)
scripts/
  fetch-sprites.ts  # הורדת ספרייטים (idempotent)
  balance.ts        # מנוע בדיקת איזון (הרצת אלפי קרבות head-less)
```

## מנגנון הקושי (DDA)
היריב נבנה לפי "תקציב כוח" דינמי שמעוגן לכוח השחקן עם ריכוך בשלבים המוקדמים ומנגנון
pity לאחר הפסדים רצופים — כדי שההתחלה תהיה קלה ומעודדת, והמשך המשחק מאתגר אך הוגן.
תוצאות `npm run balance` מאמתות עמידה ביעדי אחוזי הניצחון שהוגדרו ב-GDD §12.
