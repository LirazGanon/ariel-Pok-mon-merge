# רשימת משימות פיתוח — "אריאל: משלב הפוקימונים"

מסמך זה מפרק את [GDD.md](./GDD.md) לבֶּקְלוֹג פיתוח מקצועי, מסודר לפי Epics וסיפורים (User
Stories), עם קריטריוני קבלה (Acceptance Criteria), הערכת גודל (S/M/L/XL) וסדר עדיפויות.
מיועד לניהול כ-Issues/Board (GitHub Projects, Jira וכו').

מקרא גודל: **S** = עד יום עבודה, **M** = 2-3 ימים, **L** = שבוע, **XL** = מעל שבוע / דורש פיצול נוסף.

## סדר עדיפויות מומלץ (MVP קודם)

1. **Sprint 0** (תשתית) → **Epic 1** (Splash/תפריט) → **Epic 2** (נתונים+ספרייטים).
2. **Epic 3** (לוח+מיזוג) → **Epic 4** (חנות) — עד כאן כבר ניתן "לשחק" בשלב ההכנה.
3. **Epic 5** (מנוע קרב) — הליבה הכי מורכבת, לפצל לתת-Sprintים.
4. **Epic 6** (AI/קושי) — קריטי לחוויה, אך אפשר להתחיל עם גרסה פשוטה (בודג'ט קבוע ללא DDA) ולשפר איטרטיבית.
5. **Epic 7+8** (פרוגרסיה + גלגל מזל) — משלימים את הלולאה המלאה.
6. **Epic 9+10** (נגישות, ביצועים, ליטוש, פריסה) — MVP ציבורי.

> **המלצה מקצועית:** לבנות **Vertical Slice** ראשון (שלבים 1-3 בלבד, פוקימון בסיסי אחד מכל
> נדירות, ללא גלגל מזל) לפני הרחבת תוכן, כדי לוודא שה-Core Loop מהנה ומאוזן לפני השקעה
> בתוכן נוסף.
